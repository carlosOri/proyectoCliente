Registro clientes

INTRODUCCION:
El trabajo consiste en un formulario, se recogen los datos del formulario y se procesan, los datos serán guardados en una función la cual recibe la instancia de una clase (Cliente), esta clase hereda de otra (Persona) simplemente para manejar la herencia como hemos visto en el temario totalmente prescindible, estos datos pasarán por la función validar la cual hará ciertas validaciones se hacen en javaScript pero se pueden hacer en HTML , se elige JavaScript por practica de los conocimentos adquiridos, luego de las validaciones pasaremos a la inserción de los datos del cliente en el array que se ha creado y que se almacenará en localStorage para la persistencia de datos, una vez los datos hayan sido validados y almacenados pasaremos a la cración del HTML donde crearemos un parrafo y los botones borrar y editar que junto a los botones crear y resetar son los eventos que maneja este proyecto, se crean diferentes archivos JavaScript para tener más separado el código, se crea una pagina principal index.html desde la cual se accederá ha esta intro al formulario y a la página github por si se considera hacer alguna variación.

CONSIDERACIONES:
Se intenta cumplir los requisitos que pedia el trabajo, la mayor dificultad con la que me he encontrado es la edición de los registros y mantener la persistencia del id de los registros, tal y como esta se editan y se vuelven a insertar sin borrar el registro antiguo, si el trabajo persiste con esta funcionalidad es que no le he encontrado una solución factible, al manejar el localStorage tengo la sensación que esta parte es la que más me ha costado, del registro me quedo satisfecho ya que he podido poner en practica lo aprendido el ciclo y un poco más. La solución encontrada para la edición es que el registro se borre pero se crea un nuevo registro con un id diferente.
